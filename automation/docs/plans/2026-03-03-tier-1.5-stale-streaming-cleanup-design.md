# Tier 1.5: Stale + Streaming Cleanup

**Date:** 2026-03-03
**Status:** Approved

## Problem

Files unwatched for 90+ days that are available on streaming waste disk space and codec conversion resources. Currently, tier 1 (weekly) only deletes active streaming matches, and tier 2 (monthly) only catches 365-day stale items >3GB. There's a gap for medium-term stale items that are also on streaming.

## Solution

New tier 1.5 with two phases:
1. **Flag phase** — items unwatched 90+ days AND available on streaming get flagged
2. **Delete phase** — items flagged 15+ days ago that are still unwatched and still on streaming get deleted

Codec manager reads flags to skip stale files from conversion.

## Data Model

New column in streaming checker's `streaming_status` table:

```sql
stale_flagged_at  TEXT  -- ISO8601 timestamp when flagged (NULL = not stale)
```

Auto-migration adds the column if missing (same pattern as `streaming_seasons`).

## Subcommands

### `stale-flag`

- Scans all library items (Sonarr + Radarr)
- Queries Emby last-played via Activity Log API (`get_last_played_map()`)
- For each item:
  - Skip if keep-local or dual-audio
  - If unwatched 90+ days AND currently on streaming → set `stale_flagged_at = now()` (no-op if already flagged)
  - If flagged but watched since flagging → clear flag (user showed interest)
- Discord notification: count of newly flagged items

### `stale-delete`

- Reads all items where `stale_flagged_at` is 15+ days ago
- For each item:
  - Re-verify: still unwatched, still on streaming, not keep-local, not dual-audio, not actively playing in Emby
  - Delete via arr API (same as `confirm-delete`)
  - Emby refresh + arr rescan
- Discord notification: deleted items list (chunked, same format as other tiers)

## Codec Manager Integration

New skip reason `stale_candidate` in the exclusion chain:

```
missing → probe_failed → hdr → uhd → streaming_candidate → stale_candidate → already_compliant → ELIGIBLE
```

Implementation:
- `load_stale_candidates()` — reads paths from streaming DB where `stale_flagged_at IS NOT NULL`, loads into bash associative array `_STALE_PATHS`
- `is_stale_candidate_inline()` — pure bash lookup, no subprocess
- Applied in both `plan_cmd` and `enqueue_import_cmd`
- Fail-open: absent DB → return 1 (no skip)
- Same path-matching as streaming candidates: TV uses `${filepath%%/Season [0-9]*}`, movies use directory

## Cron Schedule

| Job | Schedule | After |
|-----|----------|-------|
| `stale-flag` | `30 5 * * 0` (Sunday 5:30 AM) | streaming scan (5 AM) |
| `stale-delete` | `30 6 * * 0` (Sunday 6:30 AM) | stale-flag, before tier 1 |

## Protections

All existing protections apply:
- Keep-local tags → skip
- Dual-audio (jpn+spa or eng+spa) → skip
- Active Emby playback → skip deletion
- Must still be on streaming at deletion time
- Re-check all protections immediately before deletion

## Flow

```
Weekly Sunday:
  05:00  streaming scan (existing)
  05:30  stale-flag: 90d unwatched + on streaming → flag
  06:30  stale-delete: flagged 15+ days → delete

Codec manager (any run):
  plan/enqueue → load_stale_candidates() → skip_reason = stale_candidate
```
